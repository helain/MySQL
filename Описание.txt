Данная база данных - представление сайта по продаже автомобилей (авто.ру)

В ней ведется учет объявлений, выложенных на сайте и актуальности данных объявлений.
Так же есть возможность отследить продажи авто, как по годам, так и по маркам и моделям авто.

Описание БД:
1. Таблицы:
 -auto (таблица с моделями авто)
 -auto_color (таблица цветов для размещения в объявлении, так же есть поле с RGB кодом для размещения на сайте, например)
 -auto_mark (справочник с марками авто)
 -auto_status (справочник состояний автомобилей)
 -companies (справочник компаний, представители которых зарегестрированы на сайте)
 -media (медиафайлы, приложенные к объявлениям (фото и видео))
 -media_types (справоник типов медиафайлов)
 -message_status (справочник статусов сообщений)
 -messages (таблица сообщений между пользователями)
 -offer_status (статусы объявлений)
 -profiles (профили пользователе - различная частная информация, а так же ссылка на профиль компании, если пользователь указан как представитель)
 -sales_offer (объявления о продаже авто)
 -user_type (справочник с типами пользователей)
 -users (таблица пользователей - логин/пароль, контакты, а так же дата рождения для проверки на совершеннолетие)
2. Процедуры:
 - actual_check (при запуске процедура проверяет актуальность объявления (сравнивает дату последнего обновления с текущей, если прошло более пяти дней - объявление теряет статус "актуально" и получает - "продается"). это необходимо для организации сортировки объявлений)
3. Представления:
 - auto_sold (сортировка проданных авто по году продажи. Может быть полезно для изучения статистики продаж)
 - auto_on_sale (сортировка действующих объявлений. Можно сортировать по статусу объявлений, маркам и моделям авто, а так же по их состоянию)
4. Триггеры:
 - company_check (проверяет наличие данных о компании при создании профиля, который указан как представитель)

Порядок работы 
 - auto_ru_DB - основной файл. Включает в себя создание таблиц с внешними ключами, триггеров, процедур, представлений и индексов.
 - auto_ru_filled - заполнение таблиц данными
 - auto_ru_indexes - отдельный файл с индексами (включен в основной файл)
 - auto_ru_proc файл с описанием процедуры (включен в основной файл)
 - auto_ru_trigger - описане триггера (в основном файле), так же есть проверка работы.
 - auto_ru_views - представления (есть в основном файле)
 - auto_ru_zaprosi - примеры завпросов к БД.
 - auto_ru_DB_model - модель БД с включенной в него ER диаграммой
 - ER_Diagram - ER-диаграмма в pdf формате (лучше развернуть на попарный просмотр страниц)
